Acerca de RefreshOS — Bodega Virtual de Bebidas

Proyecto escolar que simula una plataforma B2B para distribución al mayoreo de bebidas. Interfaz simple, gestión de pedidos por lotes y notificaciones por correo para facilitar el flujo comercial sin pagos en línea.

Ver catálogo

Misión y visión

Misión: Facilitar el abastecimiento al por mayor de bebidas para pequeños y medianos comercios mediante una plataforma clara y eficiente, enfocada en pedidos por lotes y comunicación directa.

Visión: Ser una herramienta educativa que demuestre buenas prácticas de diseño web y arquitectura de sistemas para comercio mayorista, priorizando simplicidad, trazabilidad y experiencia de usuario.

Enfoque académico

Desarrollado como proyecto escolar para demostrar integración frontend-backend, diseño de base de datos y envíos automáticos de correo.

Operación simple

Pedidos por lotes, historial de pedidos y notificaciones — sin pasarelas de pago integradas, ideal para B2B y pruebas académicas.

Privacidad mínima

Solo se almacenan datos esenciales del usuario y su historial de pedidos, acorde con el alcance del proyecto.

Arquitectura y tecnologías

El proyecto emplea un stack moderno y ligero pensado para un prototipo funcional: frontend con Tailwind + Flowbite, backend con Node.js/Express, y base de datos relacional (MySQL o MariaDB). El envío de correos se configura con un servicio SMTP (Nodemailer / SendGrid).

  • Frontend

    HTML5, Tailwind CSS, Flowbite.

  • Backend

    Node.js, rutas REST para usuarios, productos y pedidos.

  • Datos

    Modelo relacional optimizado con tablas para usuarios, productos, pedidos y relación de pedido-productos.

Equipo del proyecto

Curso: Gestión de proyectos (Semestre 7). Este prototipo tiene fines académicos y de demostración.
